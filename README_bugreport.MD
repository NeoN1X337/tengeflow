# Отчет о тестировании системы фильтрации и CRUD (README_bugreport.MD)

## Тест-кейс 1: Проверка фильтрации
**Сценарий:** Создан только доход. При переключении на "Расходы" или выборе категории, не соответствующей доходу, данные должны скрываться.

### Результаты тестирования

| Модуль | Ожидаемое поведение | Фактическое поведение | Статус |
| :--- | :--- | :--- | :--- |
| **Страница История (Transactions.jsx)** | Загрузка списка операций с фильтрами. | **Белый экран (Crash).** Ошибка React в компоненте `<Transactions>` (строка 26:37). | ❌ ПРОВАЛЕНО |
| **Фильтр "Расходы" (Dashboard)** | Скрытие доходных операций. | Доходы **продолжают отображаться**, даже когда выбран фильтр "Расходы". | ❌ ПРОВАЛЕНО |
| **Фильтр по категориям (Dashboard)** | Скрытие операций с несовпадающей категорией. | Список **не обновляется**. Операции остаются видимыми независимо от выбранной категории. | ❌ ПРОВАЛЕНО |
| **Фильтр по месяцам (Dashboard)** | Скрытие операций не из текущего месяца. | Работает корректно. При выборе пустого месяца отображается "Операций не найдено". | ✅ УСПЕШНО |

## Тест-кейс 2: Проверка CRUD операций (Dashboard)

| Действие | Ожидаемое поведение | Фактическое поведение | Статус |
| :--- | :--- | :--- | :--- |
| **Редактирование (Edit)** | Обновление данных в списке в реальном времени после сохранения. | Модальное окно закрывается (иногда со второго раза), но данные в списке остаются прежними. | ❌ ПРОВАЛЕНО |
| **Удаление (Delete)** | Вызов окна подтверждения (confirm) и удаление записи. | Полное отсутствие реакции на нажатие иконки "Корзина". | ❌ ПРОВАЛЕНО |

**Дополнительное наблюдение:** Манипуляции с данными блокируются или не отображаются из-за логического конфликта фильтрации: операции недоступны для изменения, если пользователь находится во вкладке "Расходы", которая не соответствует фактически имеющимся входным данным (доходам).

## Технический анализ (Консольные ошибки)
- **Transactions.jsx:26:37**: `Uncaught TypeError: Cannot read properties of undefined (reading 'start')`. Вероятно, `dateRange` или `filters` не инициализированы вовремя или передаются некорректно в `useTransactions`.
- **useTransactions.js**: Хотя функции `updateTransaction` и `deleteTransaction` были добавлены в код, они, похоже, не вызываются корректно из UI или имеют внутренние ошибки, препятствующие завершению операции.

---
**Важно:** Исходный код не изменялся согласно инструкции. Данный отчет предназначен для дальнейшего исправления багов.
